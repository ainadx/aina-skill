# AiNa — App Factory Skill

Turn **one request** into a complete product package — a full **document set**
(braindoc → SOW → PRD → dev notes → security/readiness → marketing → ops runbook)
plus a **working codebase** — all generated by **your own local Claude**. No
server, no required API key, no dependency on anyone else's infrastructure.

The documents are the headline: every pipeline stage leaves a detailed markdown
doc in `docs/`, so you walk away with the thinking *and* the code.

## Install

**One-liner — works in any environment (recommended):**
```bash
npx @whitenoisenpm/aina-skill@latest             # → ~/.claude/skills/aina-skill (all projects)
npx @whitenoisenpm/aina-skill@latest --project   # → ./.claude/skills/aina-skill (this repo only)
```
Restart Claude Code, then run **`/aina-skill`**. Needs only Node + python3.

**Or as a Claude Code plugin** (adds a guided config prompt + marketplace updates):
```bash
/plugin marketplace add ainadx/aina-skill
/plugin install aina-skill          # → /aina-skill:build
```

## Use

```
/aina-skill        (npm install)   ·   /aina-skill:build   (plugin install)
```
…then describe what you want to build.
You'll be asked a short intake round, shown each stage for approval, and end with:
```
aina-out/<slug>/
├── docs/                 ← the document set (the winner)
│   ├── 01-braindoc.md
│   ├── 02-intent-sow.md
│   ├── 03-ideation-prd.md
│   ├── 04-dev-notes.md
│   ├── 07-marketing.md
│   └── 09-ops-runbook.md
├── app/                  ← the generated codebase
└── README.md
```
…delivered as a folder, a `.zip`, or a pushed GitHub repo (your choice).

## Engine

- **local (default):** your Claude runs every stage. Zero config, zero API cost
  beyond your own session.
- **openrouter (optional):** set `engine = openrouter` + your OpenRouter key to
  offload the bulk analyst work to a cheaper model and save your Claude quota.

## Configuration (set on `/plugin install`)

| Option | Default | Purpose |
|--------|---------|---------|
| `engine` | `local` | `local` or `openrouter` |
| `openrouter_api_key` | — | only for `engine=openrouter` (stored in your keychain) |
| `output_mode` | `zip` | `folder` · `zip` · `git-repo` |
| `github_org` | — | owner for `git-repo` mode (blank = personal) |
| `deploy_target` | `none` | `none` · `vps` · `vercel` |
| `vps_host` | — | `user@host` of your own server (for `deploy_target=vps`) |

## Prerequisites

- **Claude Code** (the engine).
- **python3** — for the bundled helper scripts (stdlib only).
- **git** + **gh** (GitHub CLI, authenticated) — only for `output_mode=git-repo`.
- **docker**/**ssh** — only for `deploy_target=vps`.

## What's inside

- `skills/build/SKILL.md` — the orchestration brain.
- `personas/` — the staged persona prompts that produce each document.
- `starters/` — golden app starters (FastAPI + HTMX, server-rendered, can't ship bland).
- `scripts/` — self-contained stdlib helpers (classify · seed · package).
